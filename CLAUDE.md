# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**aulas-engenharia-prompt** is a static educational course on "Engenharia de Prompt em Análises Investigativas" (Prompt Engineering for Investigative Analysis). All content is authored directly in HTML files with embedded CSS and JavaScript — no build system, no external dependencies.

**Published at:** https://paulosrl.github.io/aulas-engenharia-prompt/

## Architecture

### File Structure

- **`html/`** — Lesson files (complete, self-contained HTML files)
  - `aula1.html`, `aula2.html`, `aula2-pratica.html`, `aula3.html`, `aula3-pratica.html`
  - Each file contains embedded CSS (theme colors, sidebar, layout) and JavaScript (dark mode toggle, copy-to-clipboard)
  
- **`index.html`** — Landing page with navigation links to all lessons
  
- **`dados-exemplo/`** — Markdown files with example data for exercises
  - `dados-topico1.md`, `dados-topico2.md`, `dados-topico3.md`
  
- **`graphify-out/`** — Knowledge graph visualization (auto-generated by graphify)
  
- **`README.md`** — Public documentation and editing guidelines
  
- **`AGENTS.md`** — Instructions for Claude agents (references graphify workflow)

### Design Principles

1. **HTML as Source of Truth** — All content lives in HTML files. No markdown sources locally.
2. **Offline-First** — Every HTML file is self-contained; works without internet, CDN, or external libraries.
3. **Single-File Lesson** — Each lesson is one HTML file with all CSS and JS embedded.
4. **Responsive & Accessible** — Works on desktop and mobile; includes dark/light mode toggle.

## Lesson Structure (Pedagogical Pattern)

Every lesson follows this structure:

1. **Título** — Clear, professional title
2. **Objetivo** — Learning outcomes (3–5 bullet points with ✅)
3. **Agenda** — Topics and estimated time
4. **Tópico 1, 2, 3, ...** — Each topic includes:
   - 📌 **Situação-Problema** (context and challenge)
   - 🎯 **Objetivo** (what to achieve)
   - 🔧 **Solução** (step-by-step technique)
   - 💡 **Por que Funciona** (benefits and rationale)
   - 📋 **Exemplo Prático** (worked example)
   - ✍️ **Exercício** (student assignment)
5. **Integração Final** — How techniques connect
6. **Fechamento** — Summary and next steps

## HTML Template & Conventions

### Prompt ID Convention

When adding prompts to an HTML file, use these ID patterns:

```html
<pre id="prompt-aX-Y">Your prompt text here...</pre>
<div class="prompt-actions">
    <button class="btn copy-btn" onclick="copyPrompt('prompt-aX-Y', this)">Copiar</button>
    <a class="btn gemini" target="_blank" href="https://gemini.google.com/app" onclick="copyPrompt('prompt-aX-Y')">Testar no Gemini</a>
    <a class="btn claude" target="_blank" href="https://claude.ai/new" onclick="copyPrompt('prompt-aX-Y')">Testar no Claude</a>
    <a class="btn copilot" target="_blank" href="https://copilot.microsoft.com/" onclick="copyPrompt('prompt-aX-Y')">Testar no Copilot</a>
    <a class="btn chatgpt" target="_blank" href="https://chatgpt.com/" onclick="copyPrompt('prompt-aX-Y')">Testar no ChatGPT</a>
</div>
```

**ID Naming Convention:**
- `prompt-aX-Y` — Prompts (aX = lesson X, Y = sequence number)
- `base-aX-Y` — Examples/base data
- `p1`, `p2`, `p3` — Main sections

### Theme Colors

All lessons use CSS variables defined in `:root`:

```css
:root {
    --accent: #C9A84C;        /* Gold/bronze brand color */
    --bg-main: #f3f4f6;       /* Light mode background */
    --text-main: #1e293b;     /* Light mode text */
    --card-bg: #ffffff;       /* Light mode card */
    --nav-text: #1e293b;      /* Light mode nav text */
    --table-border: #e2e8f0;  /* Light mode borders */
}

body.dark-mode {
    --bg-main: #0a1120;       /* Dark mode background */
    --text-main: #f1f5f9;     /* Dark mode text */
    /* ... dark mode overrides ... */
}
```

Do NOT use hardcoded colors; always use `var(--*)` for theme consistency.

### JavaScript Features

Each HTML file includes:

1. **Dark Mode Toggle** — `toggleDarkMode()` function
   - Persists preference to localStorage
   - Toggles `body.dark-mode` class
   
2. **Copy to Clipboard** — `copyPrompt(elementId, buttonEl)` function
   - Copies text from `<pre id="elementId">` to clipboard
   - Shows feedback: button text changes to "Copiado ✓" for 2 seconds

3. **Sidebar Navigation** — Fixed navigation with links to all lessons

## Common Development Tasks

### Editing a Lesson

1. Open the HTML file (e.g., `html/aula3.html`)
2. Find the section using Ctrl+F (search for topic title or `<h2>`)
3. Edit HTML directly (no build step required)
4. Save the file
5. **Test locally in browser:**
   - Theme toggle works (light ↔ dark)
   - Copy buttons work (check browser console for errors)
   - Layout is responsive (test desktop and mobile widths)
   - No text overflow in code blocks or tables
   - All links (internal and external) work
6. Commit: `git commit -m "update: revise Aula X Tópico Y"`
7. Push: `git push` (auto-publishes to GitHub Pages)

### Adding a New Lesson

1. Duplicate an existing lesson file (e.g., `cp html/aula2.html html/aula4.html`)
2. Update `<title>` tag and lesson title (`<h1>`)
3. Modify sidebar navigation links in the HTML (if needed)
4. Write lesson content following the pedagogical pattern above
5. Add the link to `index.html` in the lessons list
6. Update `README.md` "Status das Aulas" table
7. Test thoroughly (see "Editing a Lesson" checklist)
8. Commit and push

### Testing Offline-First Requirement

- Open the HTML file directly in a browser (no local server needed)
- Verify all functionality works without internet
- Check that embedded fonts render correctly
- Confirm dark/light mode works

## Restrictions & Constraints

### Offline-First (CRITICAL)

❌ **Do NOT use:**
- CDN links (Google Fonts, FontAwesome, Bootstrap, etc.)
- External JavaScript libraries
- External CSS frameworks
- Remote image hosting
- API calls to external services

✅ **Instead:**
- Embed all CSS in `<style>` tag
- Embed all JavaScript in `<script>` tag
- Use system fonts (fonts already installed on most machines)
- Inline images as base64 (if necessary)
- Self-host all resources

### HTML Structure

- Single file per lesson (no modularization)
- All CSS and JS in the same file
- `<head>` contains `<meta charset="UTF-8">` and viewport meta tag
- Sidebar navigation is consistent across lessons
- Use semantic HTML5 elements (`<header>`, `<main>`, `<section>`, `<article>`)

## Common Patterns

### Dark Mode Implementation

Dark mode is handled by toggling the `dark-mode` class on `<body>`:

```javascript
function toggleDarkMode() {
    document.body.classList.toggle('dark-mode');
    localStorage.setItem('darkMode', document.body.classList.contains('dark-mode'));
}

// Initialize on page load
if (localStorage.getItem('darkMode') === 'true') {
    document.body.classList.add('dark-mode');
}
```

### Copy to Clipboard

```javascript
function copyPrompt(elementId, buttonEl) {
    const element = document.getElementById(elementId);
    const text = element.innerText || element.textContent;
    navigator.clipboard.writeText(text).then(() => {
        if (buttonEl) {
            const originalText = buttonEl.textContent;
            buttonEl.textContent = 'Copiado ✓';
            setTimeout(() => {
                buttonEl.textContent = originalText;
            }, 2000);
        }
    });
}
```
